# SilentBridge 初赛 Demo · 残酷评审与晋级方案

> 评审立场：TRAE AI 创造力大赛评委（社会服务 + 社会公益）  
> 评审维度：技术可行性 · 创意创新性 · 场景落地价值 · 商业发展潜力 · 社会公益意义 · 产品表达  
> 结论：**当前是“能跑的好原型”，还不是“评委 90 秒内会记住并投晋级票的作品”。**

---

## 0. 一句话总评

**产品内核很强（真实痛点 + 可演示闭环 + 双引擎兜底），但首屏叙事、演示路径、桌面体验、情绪峰值都没打满。**  
评委不会给你 8 分钟慢慢点；他们会在 60–90 秒内判断：

1. 解决谁的什么问题？  
2. 和“普通字幕工具”差在哪？  
3. 我点一下能不能马上感受到价值？  
4. 这东西能不能进医院/政务窗口？

现在的 Demo **在第 2、3 点上危险**。

## 2. 致命问题（会直接掉分）

### P0-1. 没有“评委一键看懂”路径

现状：默认路径是「首页 → 开始 → 展示卡 → 收听 → 麦克风/手动输入 → AI 整理 → 保存」。  
评委在公共 Wi‑Fi / 不授权麦克风 / 不愿输入时，**核心价值 60 秒内出不来**。

**晋级必需：**  
顶部常驻 **「一键演示（无需麦克风）」**，45–60 秒自动：

展示开场白 → 模拟字幕逐条 → AI 重点/风险/确认问题 → 一键确认 → 摘要保存。

### P0-2. Landing 页是标准 AI 模板，不“像作品”

- 白底 + mint + 三卡片 + “准备好体验了吗”  
- 没有故事、没有对比、没有 15 秒产品动画  
- 技术栈自夸过多，用户价值过少  

评委第一眼：**又一个 AI 落地页**。

### P0-3. 手机框在桌面被浪费，宽屏没有“展台感”

`.sb-device-frame { width: min(100vw, 430px) }`  
评委用笔记本打开：中间一条手机，两侧空白。  
**像学生作业预览，不像产品 Demo。**

应做成：

- 左：痛点 → 价值 → 本场演示剧本  
- 右：可交互手机  
- 顶：一键演示 / 进入真实模式

### P0-4. 破戏文案直接减分

- 「给评委看的第一步」  
- 过多“演示”“fallback”暴露在主路径  

评委要沉浸在用户故事里，不是看你在提醒“这是比赛 Demo”。

### P0-5. “大字给对方看”不是全屏峰值

`DisplayCard` 虽字大，但仍是卡片 + 导航 + 步骤条。  
真实场景是：**把手机怼到对方眼前**。  
缺全屏 / 高对比 / 翻转友好 / 亮度感 / “对方正在看”状态。

### P0-6. AI 差异化不够“一眼可见”

产品卖点是 **理解 + 确认 + 风险**，不是转写。  
现在 AI 卡在流程后半段才出现，视觉权重弱于录音球和按钮。  
评委容易归类为：**“又一个语音转文字”**。

### P0-7. 步骤叙事被砍成 2 步

Progress 只有「给对方看 / 听对方说」。  
真正价值在第 3 步 **「小桥理解 / 确认」** 和第 4 步 **「留下重点」**。  
进度条应反映完整价值链路。

### P0-8. 报名叙事 vs Demo 场景漂移

报名帖强调：医院问诊、求职面试、课堂、政务。  
Demo 变成：药店、窗口、问路、通用。  
**医院问诊的公益冲击力最强**，被弱化了。

### P0-9. 视觉体系分裂

- Landing：Tailwind 温和 SaaS  
- Demo：新粗野主义 + 卡通吉祥物  

不是不能可爱，但 **听障就医/政务** 需要“可信工具感 + 温和无障碍”，不是“儿童教育 App”。  
吉祥物可以保留，但要降级为辅助，不当主视觉。

### P0-10. 保存后的情绪峰值不够

保存后跳到记录列表，缺少：

- “这次不会再漏掉了”  
- 可分享摘要卡  
- 一键继续追问的戏剧感  

**公益产品要让评委感到被保护。**

## 3. 中等问题（拉开分差）

1. 听桥页信息过载：录音球 + 主按钮 + 工具条 + 字幕 + 备用输入 + 权限 + 恢复 + AI + 安全条，一屏像仪表盘。  
2. 快捷短语与场景入口割裂，评委不知道先点哪。  
3. “本地规则整理 / GLM-4”对用户无意义，应换成“已整理 / 实时 AI”。技术细节放次级。  
4. 无障碍：字重 950 过多导致“喊叫感”；部分蓝字对比不足；缺系统字号放大策略。  
5. 移动端优秀，但比赛评测主要在桌面浏览器。  
6. 缺 30 秒“前后对比”：没有工具时漏掉复诊/禁忌 vs 有工具后确认完整。

---

## 4. 做得好的地方（必须保留）

1. **真实沟通流程**：展示 → 收听 → 理解 → 保存，比纯字幕高级。  
2. **失败恢复完善**：麦克风拒绝、ASR 失败、Agent 失败都有路径。  
3. **双引擎**：proxy + fallback，演示不易白屏。  
4. **记录可续聊**：会话资产化，有产品深度。  
5. **场景化抽取与风险**：药店禁忌、政务材料，贴近公益。  
6. **工程完整度**：monorepo、类型、测试、部署文档，技术可行性分有底。

## 5. 晋级方案：48 小时作战清单

### A. 必须做（决定晋级）

| # | 项 | 验收 |
|---|---|---|
| A1 | **一键演示模式** | 无麦克风 60s 跑通全链路，可暂停/重放 |
| A2 | **Landing 重做** | 首屏 5 秒讲清：听障关键对话副驾驶 + 对比 + CTA |
| A3 | **桌面展台布局** | 左故事右手机，评委笔记本第一眼完整 |
| A4 | **全屏展示卡** | 一键全屏给对方看，高对比大字 |
| A5 | **AI 结果峰值** | 已确认 / 未确认 / 风险 / 建议问题 视觉压倒字幕 |
| A6 | **去掉破戏文案** | 主路径无“给评委看” |
| A7 | **医院场景回补** | 至少 1 个强公益场景（问诊或强化药店） |

### B. 强烈建议（拉开差距）

| # | 项 |
|---|---|
| B1 | 进度条改为 展示 → 收听 → 理解 → 保存 |
| B2 | 保存成功页：摘要卡 + 分享文案 + 继续追问 |
| B3 | 首页默认引导：点「药店问药」一键进入，而非自由输入 |
| B4 | 对比模块：普通字幕 vs SilentBridge |
| B5 | 机构侧一句：医院/政务窗口可部署无障碍沟通终端 |

### C. 可延后（复赛）

- 真实医院 ASR 优化、手语、多端账号、LangGraph 可视化炫技  
- 过度 3D / 复杂动效（评委更吃清晰价值）

## 6. 评委 90 秒理想路径（目标体验）

1. 打开链接 → Landing 3 秒看懂痛点  
2. 点「立即体验」→ 进入桌面展台  
3. 点「一键演示」→ 自动药店/医院剧本  
4. 看到：大字开场白 → 字幕 → **风险高亮** → **请确认用药**  
5. 点保存 → 摘要卡：“饭后两次 / 禁酒 / 需确认药名”  
6. 评委结论：**这不是字��工具，是关键对话副驾驶。**

## 7. 文案与品牌修正

| 现在 | 改为 |
|---|---|
| 给评委看的第一步 | 删；或「从一次现场沟通开始」 |
| AI 听障沟通副驾驶（抽象） | 听不清时，先把关键的话留下来 |
| 本地规则整理 | 已整理（离线可用） |
| GLM-4 实时整理 | AI 实时整理 |
| 小桥（可保留） | 降为助手名，不当吉祥物主角 |

核心理念保持：

> **不是让听障者适应嘈杂世界，而是让世界把重要的话说清楚。**

---

## 8. 视觉方向修正（不推翻重做）

保留：高对比、圆角卡片、薄荷绿/琥珀点缀、硬边阴影（有记忆点）。  
调整：

1. 减少粉嫩渐变面积，背景更克制  
2. 主 CTA 用深色可信按钮，珊瑚做强调而非整屏玩具感  
3. 展示卡：白底黑字或黑底白字高对比模式  
4. 风险：琥珀/珊瑚强提示，不可被“可爱”稀释  
5. Landing 与 Demo 统一字体层级与色板

## 9. 功能缺口（初赛可补的最小集）

1. ✅ 已有：场景、字幕、AI、快捷语、记录  
2. ❌ 缺：**一键演示**  
3. ❌ 缺：**全屏出示**  
4. ❌ 缺：**前后对比叙事**  
5. ❌ 缺：**分享/复制摘要**  
6. ⚠️ 弱：医院场景冲击力  
7. ⚠️ 弱：桌面展台
