# SilentBridge 真功能审计（不只 90 秒演示）

> 结论先说：**上一轮主要优化的是“评委表达层”**（Landing、一键演示、桌面展台、AI 峰值视觉）。  
> **真实使用功能骨架已有，但“好用程度 / 场景深度 / 结果质量”还没打满。**  
> 时间充裕时，应该两手抓：**演示路径 + 产品本体**。

---

## 1. 功能全景：有什么 / 缺什么

| 能力 | 现状 | 评级 | 问题 |
|---|---|---:|---|
| 出示开场白 / 全屏大字 | 有，已加全屏 | B+ | 亮度/翻转友好可再加强 |
| 场景入口（药店/政务/问路） | 有 3 个 | B | 缺医院问诊冲击力；与报名帖不完全对齐 |
| 麦克风录音 + ASR | 有（录音→后端/识别链路） | B | 环境差异大；失败时路径长 |
| 浏览器语音 / 演示字幕兜底 | 有 | A- | 演示模式文案仍偏“技术说明” |
| 手动输入对方回复 | 有 | A- | 与主按钮并存时信息过载 |
| AI 理解（确认/缺失/风险/追问） | 有 | B- | **自由输入抽取质量一般**；预设已兜住 |
| 一键确认问题 / TTS 朗读 | 有 | B+ | 确认后应更像“下一轮出示”高潮 |
| 快捷话术 | 有 2 组 6 句 | C+ | 太少；未按场景分包；和开桥联动弱 |
| 会话摘要保存 | 有 localStorage | B | 摘要有时像剪贴字幕，不够结构化 |
| 记录搜索/筛选/续聊 | 有 | B+ | 保存高潮弱；缺复制/分享 |
| 失败恢复 | 有 | A- | 好，但演示模式可更干净 |
| 进度草稿恢复 | 有 | B+ | 用户无感知，可提示“继续上次” |
| 多轮同场对话 | 弱 | C | 保存后追问有，当场多轮浅 |
| 无障碍（字号/对比/焦点） | 部分 | B- | 字重过重；系统放大未系统支持 |
| 机构侧/服务方模式 | 无 | D | 复赛可做；初赛可用一句话叙事 |

## 2. 上一轮到底优化了什么

### 已优化（表达层为主）
- Landing 叙事与对比
- 60 秒一键演示
- 桌面左右展台
- 全屏出示
- AI 结果卡视觉峰值
- 进度 4 步
- 破戏文案清理
- 预设药店演示理解质量（命中预设时用精修结果）

### 基本还没动（产品本体）
- 自由文本 AI 抽取准确度
- 话术库深度与场景化
- 记录资产体验（分享/结构化/高潮）
- 当场多轮沟通
- 医院问诊场景
- 听桥页真实使用信息架构
- 无障碍系统级体验

所以你的判断是对的：**不能只盯 90 秒。**

## 3. 真功能优化优先级（时间充裕版）

### P0 — 会直接影响“这是不是真产品”
1. **自由输入 AI 理解质量**  
   - 药店/政务/交通抽取更完整短语  
   - 摘要不要只截前 42 字字幕  
2. **听桥页信息架构**  
   - 主路径：字幕 → AI 结果 → 主操作  
   - 备用输入/技术说明折叠  
3. **保存结果质量**  
   - 用 confirmed/missing/risks 生成结构化摘要  
   - 保存后高潮 + 复制摘要  
4. **一键演示收尾**  
   - 自动滚到 AI 卡  
   - 引导点保存（产品闭环完整）

### P1 — 拉开“好用”差距
5. 医院问诊场景（强公益）  
6. 话术库按场景扩展（先说明/确认/求助/结束）  
7. 场景卡片进入后默认更合适的话术  
8. 继续追问时带上父记录摘要上下文  
9. 空状态/首次引导更清楚  

### P2 — 打磨与加分
10. 系统字号/高对比模式  
11. 记录导出图片/分享文案  
12. 多轮对话时间线  
13. 机构终端叙事页  

---

## 4. 建议执行顺序（不赶工期）

**Wave 1（产品内核）**  
AI 抽取 + 保存摘要 + 听桥降噪 + 演示自动滚动  

**Wave 2（场景与内容）**  
医院场景 + 话术扩展 + 首页引导  

**Wave 3（情绪与细节）**  
保存高潮、分享、无障碍、动效克制打磨  

**Wave 4（提交）**  
部署、录屏、报名帖、证据链  

---

## 5. 验收标准（真功能，不是演示表演）

1. 用户手输一段药店对话，AI 能给出完整「用法/禁忌/缺失/风险」  
2. 不靠麦克风也能完整走通并保存一条像样记录  
3. 保存的记录看起来像“沟通资产”，不像日志碎片  
4. 话术页点一句能直接开桥，且场景匹配  
5. 失败时 2 步内能恢复，不迷路  
6. 评委路径与真实路径共用同一套产品能力，不是两套假 Demo
