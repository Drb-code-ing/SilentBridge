# TRAE IDE 初赛 Demo 开发任务

将下面任务复制给 TRAE IDE，用于创建初赛可交互 Demo。每完成一个阶段都要保存 Session ID 和截图。

## 总目标

请使用 TRAE IDE 创建一个名为 `silentbridge-demo` 的单页网页应用，用于 TRAE AI 创造力大赛初赛。作品必须能在浏览器中直接体验，展示 SilentBridge 无声桥如何帮助听障、弱听、临时失语及不便开口人群在关键场景中理解、确认和回应对话。

## 技术要求

1. 使用前端网页实现，可使用 HTML/CSS/JavaScript 或 Vite 前端项目。
2. 不依赖真实后端，第一版用本地模拟数据。
3. 页面必须可交互，不只是静态展示。
4. 保持可部署为静态站点或导出 HTML Zip。
5. 代码、样式、动效和交互均由 TRAE IDE 生成和修改。

## 产品页面结构

### 1. 顶部产品区

- 产品名：SilentBridge 无声桥
- 当前模式选择：医院问诊、求职面试、课堂会议、政务窗口
- 当前会话状态：实时辅助中

### 2. 实时字幕区

根据选择的场景，播放或逐行显示模拟对话。

医院问诊示例：
- 医生：这次主要是咽喉炎，先按这个剂量吃三天。
- 医生：如果出现胸闷、持续高烧，不能等复诊，要马上来急诊。
- 用户：我想确认一下复诊时间和用药次数。

### 3. AI 重点提炼区

根据当前场景显示结构化重点卡片。

医院问诊：
- 用药：连续服用 3 天，按医嘱剂量执行。
- 风险提醒：胸闷或持续高烧时直接前往急诊。
- 需要确认：复诊日期、每日用药次数。

### 4. 一键回复建议区

展示可复制或可点击展示给对方的回复建议。

示例：
“我听不见，可以请您帮我确认：每天吃几次、复诊是哪一天、什么情况需要马上来医院？”

### 5. 快捷沟通卡

按钮：
- 请说慢一点
- 请写下关键词
- 我需要确认时间
- 我听不见，但可以通过文字沟通

点击后把对应文案显示到“给对方看的文字卡片”区域。

### 6. 会话摘要

点击“生成摘要”后，展示会话摘要卡片，包括：
- 场景
- 重点信息
- 待确认事项
- 下一步行动

## 交互要求

1. 切换场景时，字幕、重点卡片、回复建议、摘要都要同步变化。
2. 点击“开始模拟”后，字幕逐条出现，带轻量动画。
3. 点击快捷沟通卡后，页面显示一张大字卡片，适合展示给对方。
4. 点击“生成摘要”后，出现摘要卡片。
5. 移动端可用，文字不能溢出按钮或卡片。

## 视觉方向

- 参考 motion design tool / product launch page 的感觉。
- 采用白底或浅色底 + 黑色大字 + 薄荷绿 + 琥珀黄。
- 避免传统 AI 蓝紫渐变。
- 要有产品工具感，而不是公益报告页。
- 保持无障碍：清晰对比、可读字号、明显 focus 状态。
