# TRAE Work 报名 HTML 最终提示词

将下面提示词复制到 TRAE Work。目标是让 TRAE Work 生成最终可提交的报名创意方案 HTML。

```text
请为 TRAE AI 创造力大赛生成一份可直接作为报名附件上传的单文件 HTML 创意方案页面。

重要要求：
1. 作品必须体现由 TRAE Work 生成，请输出完整 HTML。
2. 页面使用内联 CSS 和少量原生 JS，不依赖外部 CDN，不引用第三方图片。
3. 页面需要有现代、精致、有运动设计感的视觉效果，避免常见 AI 模板感。
4. 页面可以参考 motion design tool / product launch page 的感觉：大标题、强对比、动态字幕条、产品界面预览、滚动入场动画。
5. 不要做成普通报告页，也不要过度煽情。要强调尊严、独立、平等参与和数字无障碍。

作品名称：
SilentBridge 无声桥——AI 听障沟通副驾驶

参赛信息：
主赛道：社会服务赛道
附加赛题：社会公益

一句话介绍：
SilentBridge 无声桥是一款面向听障、弱听、临时失语及不便开口人群的 AI 沟通副驾驶。它在医院问诊、求职面试、课堂会议、政务窗口等关键场景中，将语音对话实时转写成清晰字幕，自动提炼重点，并生成自然、得体、可一键展示给对方的回复建议，让每一句重要的话都不被错过。

页面风格要求：
- 首屏要非常有辨识度，避免传统 AI 蓝紫渐变。
- 视觉方向：白底或浅色底 + 黑色大字 + 薄荷绿 + 琥珀黄 + 少量高对比几何块。
- 首屏大标题：SilentBridge / 无声桥。
- 右侧或下方放一个“动态沟通画布”的产品界面示意：实时字幕流、AI 重点卡片、一键回复建议、沟通快捷卡。
- 加入横向滚动 ticker，例如 REAL-TIME CAPTION / KEY MOMENTS / REPLY DRAFT / PUBLIC GOOD / ACCESSIBILITY。
- 加入轻量滚动入场动画：标题、卡片、功能块进入视口时淡入和滑入。
- 页面必须响应式适配手机端。
- 保留 prefers-reduced-motion 降级。

页面内容结构：

1. 首屏 Hero
   - 标题：SilentBridge 无声桥
   - 标签：TRAE AI 创造力大赛 · 社会服务赛道
   - 副标题：给听障、弱听、临时失语及不便开口的人，一个能看懂重点、确认下一步、从容回应的 AI 沟通副驾驶。
   - CTA：看 Demo 场景 / 社会公益附加赛题
   - 产品界面示意包含：
     - 医院问诊模式 · 实时辅助中
     - 实时字幕：
       医生：这次主要是咽喉炎，先按这个剂量吃三天。
       医生：如果出现胸闷、持续高烧，不能等复诊，要马上来急诊。
       用户：我想确认一下复诊时间和用药次数。
     - AI 重点提炼：
       药物连续服用 3 天；胸闷或持续高烧时前往急诊；离开前确认复诊日期和每日用药次数。
     - 一键回复建议：
       “我听不见，可以请您帮我确认：每天吃几次、复诊是哪一天、什么情况需要马上来医院？”
     - 快捷卡：
       请说慢一点 / 请写关键词 / 我需要确认时间

2. 真实问题
   标题：最难的不是听不见，而是在关键时刻不敢确认。
   内容：普通字幕解决的是“看到文字”。SilentBridge 要解决的是：用户是否真的理解了对方的重点，是否知道下一步该做什么，是否能用得体的方式继续表达。
   场景卡：
   - 医院问诊：医嘱只说一次，错过就可能影响安全。
   - 求职面试：听清问题只是第一步，理解问题意图更关键。
   - 课堂会议：多人发言和快速讨论，让字幕很快变成信息噪声。
   - 政务窗口：材料、流程、地点和时限，任何一个没听清都可能白跑。

3. 核心功能
   标题：从语音转文字，升级为沟通理解与表达辅助。
   功能块：
   - 场景化实时字幕
   - 重点提炼卡片
   - 一键回复建议
   - 沟通快捷卡
   - 会话结束摘要

4. 初赛 Demo 场景
   标题：1 分钟内让评委看懂：这不是字幕工具，是沟通副驾驶。
   Demo 卡：
   - 医院问诊：转写医生说明，提炼用药、复诊、异常处理，并生成“请医生确认”的回复。
   - 求职面试：识别面试问题意图，生成回答提纲和自然回复，降低临场压力。
   - 课堂会议：整理多人发言，生成重点笔记、结论和待办，方便课后复盘。
   - 政务窗口：把窗口说明转换为材料清单、办理步骤和下一步确认问题。

5. 社会公益附加赛题契合点
   标题：无障碍沟通不该是额外照顾，而应是公共服务默认具备的能力。
   内容：SilentBridge 关注听障、弱听、临时失语及不便开口人群在公共生活中的沟通权利。它的公益价值不止停留在叙事上，而是可以被体验、被部署、被复用的 AI 工具。
   三点：
   - 降低特殊群体参与社会的隐形门槛
   - 提升公共服务的无障碍能力
   - 推动数字无障碍从口号变成产品

6. 后续计划
   - Phase 1 报名：创意方案 HTML
   - Phase 2 初赛：可交互网页 Demo
   - Phase 3 复赛：真实能力增强，包括语音识别、场景模板、摘要导出和无障碍交互

结尾文案：
不是让听障者适应嘈杂世界，而是让世界学会把重要的话说清楚。

请输出完整 HTML 文件，不要只输出说明。
```
