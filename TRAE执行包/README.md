# SilentBridge TRAE 执行包

这个目录用于保证后续参赛材料符合“作品需全部使用 TRAE Work 或 TRAE IDE 创作”的要求。

## 核心原则

1. 最终提交的 HTML、Demo、代码和可体验成果必须由 TRAE Work 或 TRAE IDE 生成、修改和导出。
2. 所有前期参考内容都需要转写为 TRAE 可执行提示词，再由 TRAE 完成实际创作。
3. 每个关键阶段都要保留能证明创作过程真实的材料，包括 AI 对话截图、任务历史、Session ID、阶段成果截图。
4. 对外提交材料只呈现 TRAE 创作过程、项目成果和独立创作者信息。

## 推荐流程

1. 使用 `01_TRAE_Work报名HTML最终提示词.md` 在 TRAE Work 中生成报名创意 HTML。
2. 将 TRAE Work 生成的 HTML 保存为最终提交附件。
3. 使用 `02_TRAE_IDE初赛Demo开发任务.md` 在 TRAE IDE 中创建初赛 Demo。
4. 每完成一个阶段，填写 `03_证据链记录表.md`。
5. 发帖或提交前，逐项检查 `04_合规检查清单.md`。

## 当前本地文件定位

- `报名材料/SilentBridge_创意方案.html`：前期参考稿，不建议直接作为最终附件提交。
- `报名材料/报名帖模板版_可直接发布.md`：报名帖正文参考，可复制到社区后按实际 TRAE 生成附件调整。
- `TRAE执行包/`：用于把参考稿转化为 TRAE 可执行流程。
