## A. 部署后端云函数（先做这个）

### A1. 创建函数

1. 打开：https://console.cloud.tencent.com/scf  
2. 新建函数  
3. 选：
   - 地域：建议 **靠近你用户**（如广州/上海）
   - 运行环境：**Node.js 18** 或更高
   - 创建方式：**本地上传文件夹** / 本地上传 zip  
4. 执行方法填：

```text
index.main
```

（文件 `index.js` 里导出的是 `exports.main`）

### A2. 上传代码

把这个文件夹打成 zip 或直接上传目录：

```text
E:\SilentBridge\domestic-full\scf-api\
  index.js
  package.json
```

### A3. 环境变量（必配，和 Vercel 一样）

在函数配置 → 环境变量 添加：

| 键 | 值 |
|---|---|
| `ZHIPU_API_KEY` | 你的智谱 Key |
| `BAIDU_API_KEY` | 你的百度 API Key |
| `BAIDU_SECRET_KEY` | 你的百度 Secret Key |

### A4. 触发器（HTTP 访问）

函数配置 → 触发器 → 添加触发器：

- 类型：**API 网关触发器** 或 **HTTP 访问服务**
- 发布为可公网访问  
- 记下触发地址，例如：

```text
https://service-xxxxx.gz.apigw.tencentcs.com/release
```

或

```text
https://xxxxx.scf.tencentcs.com
```

### A5. 测后端

浏览器或命令行访问：

```text
https://你的云函数域名/api/health
```

成功应类似：

```json
{"ok":true,"runtime":"tencent-scf","hasZhipuKey":true,"hasBaiduAsr":true}
```

再测（可选）：

```bash
curl -X POST "https://你的云函数域名/api/agent/run" ^
  -H "Content-Type: application/json" ^
  -d "{\"sessionId\":\"t1\",\"flowId\":\"clinic\",\"transcript\":[{\"id\":\"1\",\"speaker\":\"医生\",\"text\":\"咽喉炎，连吃三天，胸闷高烧急诊\",\"time\":\"00:01\"}],\"userMessage\":\"请确认\",\"round\":1}"
```

## C. 让前端像 Vercel 一样调用后端

纯静态站**没有**同源 `/api`。有两种接法：

### 方案 C1（推荐，最接近 Vercel）：API 网关自定义域名 + 反代/路径合并

目标最终变成：

```text
https://你的国内域名/          → 静态网页
https://你的国内域名/api/*     → 云函数
```

在腾讯云 API 网关 / CloudBase 路由里配置：

- `/api/*` → 云函数  
- `/*` → 静态托管  

这样前端代码**不用改**，仍然请求 `/api/transcribe`、`/api/agent/run`。

### 方案 C2（更快但要改一次前端）：静态站 + 绝对 API 地址

如果暂时做不到同源反代：

1. 云函数已有公网地址 `https://service-xxx...`  
2. 我可以改前端，支持环境变量：

```text
VITE_API_BASE=https://service-xxx...
```

3. 重新 build 后再上传 `web/`

你拿到云函数域名后回我，我帮你改并重新打包。
