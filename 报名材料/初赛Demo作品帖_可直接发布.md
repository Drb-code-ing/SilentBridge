### 0. 一句话

**SilentBridge 无声桥** 是一个面向听障、弱听、临时失语及不便开口人群的 **AI 现场沟通副驾驶网页 Demo**。  
它不只是字幕工具，而是把对方的话变成：**字幕 → 重点 → 风险 → 可出示的确认问题 → 可保存摘要**。

> 核心理念：不是让听障者适应嘈杂世界，而是让世界把重要的话说清楚。

### 1. Demo 简介

#### 是什么

- **产品形态**：可公开访问的 **Web 应用 / 移动端优先网页 Demo**（React + Vite + 可选后端 API）
- **使用方式**：手机递给对方看开场白 → 收听对方说话 → AI 整理重点 → 保存摘要 / 继续追问
- **技术底座**：前端 React 18 + TypeScript + Tailwind；后端 Hono；AI 可用智谱 GLM-4-Flash，失败自动降级到本地规则引擎

#### 面向谁

- **核心用户**：听障、弱听人群  
- **扩展用户**：临时失语、嗓音受限、嘈杂环境中听不清关键信息的人  
- **场景方**：医院门诊、药店、政务窗口、临时问路等公共服务现场

#### 主要功能（3 个核心）

**① 场景化现场沟通闭环**  
支持：医院问诊、药店问药、政务窗口、临时问路、通用沟通。  
流程固定为：

1. 出示开场白（可全屏大字）  
2. 收听 / 手动输入 / 演示字幕  
3. AI 提炼：已确认 / 待确认 / 风险  
4. 一键生成「给对方看」的确认问题  
5. 保存本地摘要，支持继续追问  

**② 理解比转写更重要**  
普通字幕只回答「对方说了什么字」。  
SilentBridge 额外回答：

- 哪些是必须记住的重点  
- 哪些信息还没确认  
- 有没有高风险（如急诊红线、用药禁忌、坐反方向）  
- 现在该向对方确认哪一句话  

**③ 失败也不白屏**  
真实现场很脏：麦克风拒绝、ASR 失败、网络不稳、LLM 超时。  
Demo 内置：

- 麦克风权限引导  
- 手动输入兜底  
- 演示字幕路径  
- 本地规则引擎降级  
- 记录本地持久化（`localStorage`）

### 2. Demo 创作思路

#### 灵感来源

我注意到：很多 AI 产品都在帮「听得见的人」更快写代码、更快办公，但听障用户真正卡住的，往往是日常生活里那几分钟——

- 医生交代用药和复诊  
- 窗口人员快速报材料  
- 路人随口指路  
- 临时说不清、来不及记  

这些时刻对普通人只是一次对话，对听障用户可能决定：  
**一次就医是否清楚、一次办事是否白跑、一次出行是否走错。**

#### 想解决的问题

真实痛点不是「没有字幕」，而是：

1. **信息只说一遍**，错过就很难完整追问  
2. **字幕不等于理解**，用户仍不知道什么最重要  
3. **临场压力大**，既要看字又要组织回复  
4. **服务方不是不配合**，而是缺少高效确认工具  

所以我们不做“又一个语音转文字”，而做 **现场沟通副驾驶**：

- 帮用户听清  
- 帮用户看懂  
- 帮用户敢确认  
- 帮用户把结果留下  

#### 为什么做这个方向

初赛阶段我做了三个取舍：

1. **小切口，深闭环**  
   不先做大平台，先把「出示 → 收听 → 理解 → 保存」跑通。

2. **现场可用性优先于炫技**  
   麦克风失败也要有路；LLM 失败也要有本地整理；保存后刷新还在。

3. **默认不误判场景**  
   用户没选医院，就不能默认塞医嘱剧本。冷启动是「通用沟通」，医院只在主动选择后进入。

这个方向同时契合：

- 主赛道：**社会服务**  
- 附加：**社会公益 / 数字无障碍**

### 4. TRAE 实践过程

本项目从创意到可交互 Demo，主开发链路在 **TRAE IDE / TRAE Work** 中完成：任务拆解、代码生成、页面重构、功能迭代、验收截图与 Session 留存。  
后续优化阶段继续用 AI 编程助手做审查、回归测试与交互修补，但 **初赛证据链以 TRAE Session ID + 开发截图为主**。

#### 4.1 完整流程（从 0 到可体验）

```text
报名创意梳理（TRAE Work）
    ↓
项目骨架 monorepo（pnpm + Vite + React + shared 类型）
    ↓
场景数据与沟通引擎
    ↓
移动端现场流程重设计（出示 / 收听 / 记录 / 话术）
    ↓
AI 理解环（重点 / 风险 / 确认问题）
    ↓
真实输入与麦克风 / ASR 试点
    ↓
失败恢复、权限引导、记录持久化
    ↓
初赛打磨：默认场景修正、停止识别边界、摘要保存加固
```

#### 4.2 我实际怎么用 TRAE 干活

1. **先写清任务边界**  
   每个 Phase 只做一件可验收的事，例如“先让字幕逐条出现”，而不是一次生成整站。

2. **用 TRAE 生成可运行代码，而不是只聊天**  
   从目录结构、组件拆分、样式、状态流，到 typecheck / build，都在 TRAE 会话里闭环。

3. **每阶段强制留证据**  
   - 过程截图  
   - Session ID  
   - 阶段完成摘要  
   避免最后补材料。

4. **用“可演示路径”倒逼功能**  
   先保证评委 / 用户 1 分钟点得通，再补工程细节。

#### 4.3 使用的 TRAE 能力 / Skill / MCP 工具（实践说明）

> 说明：以下按我们真实工作流整理，发布时可按社区规范附截图。名称以你本机 TRAE 中实际启用的为准。

**A. TRAE 内建能力**

- **SOLO / Agent 任务模式**：分阶段生成项目骨架、页面、状态逻辑  
- **代码编辑与多文件重构**：Bridge / Home / Records 组件拆分  
- **终端执行**：`pnpm install` / `typecheck` / `build` / `dev`  
- **预览与问题回修**：根据页面反馈改布局、字号、按钮可达性  

**B. 开发中用到的 Skill 思路（可对应你 TRAE 中的技能配置）**

- **前端界面生成 / 重构 skill**：快速搭移动端沟通主界面  
- **无障碍与可读性检查思路**：大字号、高对比、全屏出示  
- **测试与回归 skill**：对收听状态机、记录持久化、场景推断补单测  
- **文档共创 skill**：报名创意 HTML、证据链文档、部署清单  

**C. MCP / 外部工具协作（用于提效与验收）**

- **文件系统 MCP**：批量查看/修改 `apps/web` 源码与证据目录  
- **浏览器自动化（Playwright）**：端到端验收  
  - 冷启动是否误进医院  
  - 停止并识别是否产出字幕+摘要  
  - 保存后刷新记录是否还在  
- **本地 API 联调**：Hono 代理、ASR / Agent 降级路径  

这些工具不是为了堆名词，而是保证：

- 功能真能跑  
- 失败有退路  
- 证据链完整

### 5. 技术实现摘要（加分项，可选保留）

- **前端**：React 18、TypeScript、Vite、Tailwind  
- **后端**：Hono（Agent / Transcribe 代理）

- **ASR**：录音 + 服务端识别；失败可手动输入 / 演示字幕  
- **工程**：pnpm monorepo、单测、Playwright 端到端验收、记录本地持久化  
- **部署**：Vercel（`silentbridge-demo` 为 Root）

与“普通字幕工具”的差异一句话：

> 我们交付的不是转写结果，而是一次可确认、可保存、可继续的沟通结果。

### 6. 开发心得（可选）

1. **先做现场闭环，再谈完整平台。**  
2. **无障碍产品首先要“失败可恢复”。** 麦克风和模型都会挂，产品不能挂。  
3. **默认值是产品价值观。** 没选医院就不要冒充医院。  
4. **TRAE 最适合分阶段推进**：每个 Session 对应可截图、可验收的增量，证据链自然完整。
