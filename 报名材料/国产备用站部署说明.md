# 国产/国内可访问备用站方案

Vercel（`*.vercel.app`）在国内常需代理。下面给你 **主站 + 国内备用站** 的稳妥组合。

## 推荐结构

| 链接 | 用途 |
|---|---|
| **主站** https://silent-bridge-sigma.vercel.app | 完整能力：真麦克风识别 + 智谱 AI（有 Key） |
| **备用站**（国内托管静态前端） | 评委在国内稳定打开；用「演示字幕 / 手动输入」完整体验闭环 |

> 备用站一般是**纯前端静态站**：不自带 `/api`。  
> 产品本身已支持失败降级：演示字幕、手动输入、本地规则整理、记录本地保存，**足够初赛演示**。

---

## 最快方案：腾讯云 CloudBase 静态网站托管（推荐）

适合国内访问，步骤短。

### 1. 准备好的本地产物

仓库已构建好前端，目录：

```text
E:\SilentBridge\domestic-mirror\
  index.html
  assets\
```

这就是要上传的全部文件（约 300KB 级）。

### 2. 开通并创建

1. 打开：https://console.cloud.tencent.com/tcb  
2. 登录（可用微信/QQ）  
3. 新建环境（按提示开通，个人试用通常有免费额度）  
4. 进入环境 → **静态网站托管** → 开通  

### 3. 上传

1. 静态网站托管 → **文件管理**  
2. 上传 `domestic-mirror` 里的：
   - `index.html`
   - 整个 `assets` 文件夹  
3. 等待部署完成  

### 4. 拿到默认域名

页面会给出类似：

```text
https://xxxxx.tcloudbaseapp.com
```

或 CloudBase 默认静态域名。  
**这就是你的国内备用链接。**

### 5. 自测清单

打开备用域名后确认：

1. 首页能开  
2. 点「医院问诊」  
3. 点「对方看完了，开始收听」  
4. 点 **「演示字幕」**（不要依赖麦克风）  
5. 能看到重点/风险  
6. 能保存摘要，刷新后记录还在  

真麦克风在备用站可能失败（没 API），**没关系**，演示字幕路径就是给评委用的。

## 备选方案

### A. 阿里云 OSS 静态网站

1. https://oss.console.aliyun.com  
2. 建 Bucket（读权限：公共读）  
3. 开启「静态页面」：默认首页 `index.html`  
4. 上传 `domestic-mirror` 全部文件  
5. 用 OSS 默认域名或绑定 CDN 域名  

### B. 又拍云 / 七牛静态托管

同样上传 `index.html` + `assets`，开启 HTTPS 默认域名。

### C. GitHub Pages（国内一般仍不稳）

不推荐当国内备用。

## 重新打包备用站（代码更新后）

在项目里执行：

```bash
cd silentbridge-demo
pnpm --filter @silentbridge/web build
```

然后把 `apps/web/dist/*` 再上传覆盖到国内托管即可。  
（需要时也可继续用仓库里的 `domestic-mirror/` 目录作为上传包。）
