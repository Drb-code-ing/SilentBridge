## 1. Demo 简介

**SilentBridge 无声桥** 是面向听障、弱听、临时失语人群的 **AI 现场沟通副驾驶网页应用**。

不是又一个字幕工具，而是把对方的话变成：

**字幕 → 重点 → 风险 → 确认问题 → 可保存摘要**

**面向谁：** 听障/弱听用户，以及医院、药店、政务窗口、问路等公共服务现场。

**核心功能：**

1. **现场闭环**：出示开场白 → 收听/手动输入/演示字幕 → AI 整理 → 保存或继续追问  
2. **理解优先**：标出已确认、待确认、高风险（急诊红线、用药禁忌等）和可给对方看的确认句  
3. **失败可恢复**：麦克风/识别失败可降级；摘要本地保存，刷新仍在

## 2. 创作思路

听障用户真正卡住的，往往是生活里那几分钟：医嘱、窗口材料、临时指路。痛点不是“没有字幕”，而是听不清、记不住、不敢确认、事后对不上。

所以做成现场副驾驶：帮用户听清、看懂、敢确认、把结果留下。  
取舍是：小切口深闭环、可用性优先于炫技、没选医院就不默认医院场景。

契合 **社会服务** 与 **社会公益 / 数字无障碍**。

## 3. 体验地址（怎么点）

**打开这个链接：**  
https://silent-bridge-sigma.vercel.app

### 推荐演示（约 1 分钟，按按钮点）

1. 打开链接后，先看到首页「开始一次沟通」和几个场景卡片。  
2. 用鼠标/手指点 **「医院问诊」** 卡片。  
3. 进入沟通页后，中间是给对方看的开场白；可点 **「全屏出示」**（如果有）把字放大。  
4. 点 **「对方看完了，开始收听」**。  
5. 若浏览器问麦克风权限：  
   - 想试真语音：点允许，对着麦克风说话，再点 **「停止并识别」**；  
   - 不想用麦克风：点 **「演示字幕」**（或展开「打不开麦克风时」→ 填入演示 → 整理回复）。  
6. 下方会出现字幕，以及 **已确认 / 待确认 / 风险** 和确认问题。  
7. 可点 **「保存这次重点」**；不想保存可点 **「重新收听」** 或 **「改文字再整理」**。  
8. 点底部导航 **「记录」**，能看到刚保存的摘要；刷新网页后摘要还在。

**另一条路径：** 首页不选医院，直接点底部 **「沟通」** → 进入通用沟通（不会默认成医院）。

## 5. 技术与差异

React + TypeScript + Vite；部署于 Vercel；智谱 GLM-4-Flash + 百度语音；失败自动降级。

**与普通字幕的区别：** 交付的不是转写结果，而是可确认、可保存、可继续的沟通结果。
