## 1. Demo 简介

**SilentBridge 无声桥** 是一款可在线体验的 **AI 听障现场沟通副驾驶网页应用**。

它面向听障、弱听、临时失语及不便开口的人群，在医院问诊、药店问药、政务窗口、临时问路等关键场合，把对方的话变成：

**字幕 → 重点 → 风险提醒 → 可出示的确认问题 → 可保存摘要**

核心理念：不是让听障者适应嘈杂世界，而是让世界把重要的话说清楚。

### 面向谁

- 听障、弱听用户
- 临时失语、嗓音受限、嘈杂环境中听不清关键信息的人
- 医院、药店、政务窗口等需要快速确认信息的服务场景

### 主要功能

**① 现场沟通闭环**  
出示开场白（可全屏大字）→ 收听 / 手动输入 / 演示字幕 → AI 整理重点 → 保存摘要或继续追问。  
支持医院问诊、药店问药、政务窗口、临时问路与通用沟通。

**② 理解比转写更重要**  
不只显示“对方说了什么字”，还会标出：
- 已确认的关键信息
- 还没确认的事项
- 高风险提醒（如急诊红线、用药禁忌、坐反方向）
- 一句可直接给对方看的确认问题

**③ 失败可恢复**  
麦克风拒绝、识别失败、网络不稳时，仍可手动输入、演示字幕或本地规则整理；摘要可本地保存，刷新后仍在。

## 2. Demo 创作思路

### 灵感来源

很多 AI 产品都在帮“听得见的人”提效，但听障用户真正卡住的，往往是生活里那几分钟：医生交代用药、窗口快速报材料、路人随口指路。这些时刻对普通人只是一次对话，对听障用户可能决定就医是否清楚、办事是否白跑、出行是否走错。

### 想解决的问题

痛点不是“没有字幕”，而是：

1. 关键信息往往只说一遍，错过难追问  
2. 字幕不等于理解，用户仍不知道什么最重要  
3. 临场压力大，既要看字又要组织回复  
4. 服务方不是不配合，而是缺少高效确认工具  

因此我们不做又一个语音转文字，而做**现场沟通副驾驶**：帮用户听清、看懂、敢确认、把结果留下。

### 为什么做这个方向

1. **小切口，深闭环**：先跑通“出示 → 收听 → 理解 → 保存”。  
2. **现场可用性优先**：麦克风和模型都会失败，产品必须有退路。  
3. **默认值要对**：用户没选医院，就不默认塞医嘱剧本；冷启动是通用沟通，医院需主动选择。

方向同时契合 **社会服务** 主赛道与 **社会公益 / 数字无障碍** 附加方向。

## 3. Demo 体验地址（逐步点击说明）

**在线地址：**  
https://silent-bridge-sigma.vercel.app

**健康检查（可选）：**  
https://silent-bridge-sigma.vercel.app/api/health  
正常时应看到带 `"ok":true` 的 JSON。

### 怎么演示（按按钮点，约 1 分钟）

1. 浏览器打开上面的体验地址。  
2. 首页标题附近是「开始一次沟通」，下面有场景卡片。  
3. 点 **「医院问诊」** 卡片（不要先点底部「沟通」，除非你要试通用场景）。  
4. 进入后，大字区域是开场白；可点 **「全屏出示」** 放大给对方看。  
5. 点主按钮 **「对方看完了，开始收听」**。  
6. 进入收听页：  
   - **试真麦克风：** 点「开始收听」→ 允许权限 → 说话 → 点「停止并识别」  
   - **不方便说话/麦克风：** 点「演示字幕」，或展开「打不开麦克风时」→「填入演示」→「整理回复」  
7. 页面出现字幕后，往下看 **已确认 / 待确认 / 风险** 和确认问题。  
8. 可选：  
   - **「保存这次重点」** → 底部点「记录」查看，刷新网页后还在  
   - **「重新收听」** → 同页再录，不用新开沟通  
   - **「改文字再整理」** → 改识别文字后再整理  
9. 换场景：回首页再点「药店问药 / 政务窗口 / 临时问路」。

## 5. 技术与差异

- 前端：React 18 + TypeScript + Vite + Tailwind  
- 后端：Vercel Serverless（语音识别 / AI 整理代理）  
- AI：智谱 GLM-4-Flash；失败自动本地规则降级  
- 输入：麦克风录音识别 + 手动输入 + 演示字幕  
- 部署：https://silent-bridge-sigma.vercel.app  

与普通字幕工具的区别：  
**交付的不是转写结果，而是一次可确认、可保存、可继续的沟通结果。**

## 6. 开发心得

1. 先做现场闭环，再谈完整平台。  
2. 无障碍产品首先要“失败可恢复”。  
3. 默认值是产品价值观：没选医院，就不要冒充医院。  
4. TRAE 适合分阶段推进，证据链自然完整。
